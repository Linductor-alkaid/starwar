## 🚀 快速开始

### 1. 安装依赖

```bash
cd frontend
npm install
# 或
yarn install
```

### 2. 创建环境变量文件

创建 `.env.development` 文件（开发环境）：

```env
VUE_APP_API_BASE_URL=http://localhost:8080/api/v1
VUE_APP_USE_MOCK=true
```

创建 `.env.production` 文件（生产环境）：

```env
VUE_APP_API_BASE_URL=https://your-api-domain.com/api/v1
VUE_APP_USE_MOCK=false
```

### 3. 启动开发服务器

```bash
npm run serve
# 或
yarn serve
```

访问 http://localhost:8081

## 📝 功能模块

### 1. 用户认证
- 登录/注册：`/login` 或 `/register`（同一页面，通过标签页切换）
- 个人中心：`/profile`

### 2. 游戏功能
- 游戏页面：`/game`
- 排行榜：`/leaderboard`

### 3. 讨论区
- 讨论区列表：`/forum`
- 帖子详情：`/forum/:id`

## 🔧 技术栈

- **Vue 2.6.14** - 前端框架
- **Vue Router 3.5.4** - 路由管理
- **Vuex 3.6.2** - 状态管理
- **Element UI 2.15.13** - UI组件库
- **Axios 0.27.2** - HTTP客户端
- **Mock.js 1.1.0** - Mock数据

## 📌 重要说明

1. **Mock数据**：开发环境下，如果后端接口未就绪，可以启用Mock数据。在 `.env.development` 中设置 `VUE_APP_USE_MOCK=true`。

2. **API接口**：所有API接口定义在 `src/api/` 目录下，按照功能模块划分。

3. **路由守卫**：已在 `src/router/index.js` 中配置路由守卫，需要登录的页面会自动跳转到登录页。

4. **Token管理**：Token存储在localStorage中，通过 `src/utils/auth.js` 统一管理。

5. **状态管理**：使用Vuex管理全局状态，包括用户信息和游戏数据。
